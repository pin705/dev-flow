# Privacy and Redaction

## Default posture

- local-first review
- no full-repo upload by default
- selected artifact sync only
- explicit workspace-level cloud sync controls

## Admin controls

- turn cloud sync on or off
- require redaction before sync
- restrict allowed providers and models
- configure retention windows for audit history

## Developer expectations

Always assume policy metadata, trace IDs, and selected findings may sync when cloud history is enabled.
