# Auth and Doctor Troubleshooting

## Common checks

```bash
devflow auth login
devflow doctor
devflow review --staged
```

## If login fails

- confirm the browser reached the device approval page
- confirm the device session was explicitly approved
- confirm the workspace was selected
- confirm the control plane URL is correct

## If the review is empty

- make sure you have local changes
- rerun with `--staged` if changes were staged
- rerun with `--base origin/main` for branch compare

## If Qwen is missing

`devflow doctor` should report a warning and keep the CLI in scaffold mode until Qwen Code is installed.
